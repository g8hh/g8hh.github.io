# Web游戏汉化翻译

用于 `idle-artisan2-chs.user.js`（汉化插件）格式的 Web 游戏翻译。

## 执行流程

### 1. 分析 + 翻译（一次性完成）

逐条判断：
- **含动态值**（数字随游戏状态变化：速率、数量、等级、计时、百分比、加成值等，不包含数值的词句需排除） → 正则格式
- **>60字符的完整句子/段落** → 长文本格式
- **其余**（含静态数字：固定阈值、版本号、专有名称中的数字等） → 普通格式

判断是否为动态值的依据：数字是否表示玩家当前状态（资源速率、剩余数量、伤害值、冷却时间等）。格式如 `X: Y`、`X Rate: Y`、`X: Y/s`、`X Rate: Y%` 的，即使出现在词条介绍中，也视为动态值→正则格式。

固定描述中的数字（"一天30小时"、"Tier 0"、"5个槽位"）视为普通文本。

翻译时遵循：全角中文标点、术语与已有翻译一致、变量不硬编码。

### 3. 输出

按类别分组写入 `AI翻译文本.json`，结构如下：

```json
{
  "普通翻译": {
    "原文": "译文",
    ...
    "": ""
  },
  "长文本翻译": {
    "原文":
      "译文",
    ...
    "": ""
  },
  "正则翻译": [
    [/^pattern$/, ' 'replacement''],
    ...
    ["", ""]
  '],
}
```

每个类别最后添加一条空的翻译结果（`"": ""` 或 `["", ""]`）。

**正则格式规则：**
- 正则原文：使用 JS 字面量格式 `/^pattern$/`，**不要用 `"` 包裹**，即写成 `[/^pattern$/, ...]` 而非 `["/^pattern$/", ...]`
- 译文：使用**单引号 `'`** 包裹，即写成 `'replacement'` 而非 `"replacement"`
- 反斜杠无需双重转义：`\d+` 直接写，不要写成 `\\d+`
- **转义规则：** 除变量占位符（`\d+`、`\d\.,`、`\d\.`、`\.+` 等）和字母数字/中文/空格/下划线外，**所有符号都要加 `\` 转义**。常见需转义字符：`` / " . + * ? ( ) [ ] { } ^ $ | - ~ ``。注意 `/` 作为定界符也需要转义。例如原文 `"+5/s?` 写为 `/^\+(\d+)\/s\?$/`。

三类按 `普通翻译 → 长文本翻译 → 正则翻译` 顺序排列，各类内部按原文排序。已有文件时，新条目插入到对应类别区段内。

输出结果最后加上：您可以要求我读取游戏源文件或已有的翻译内容，进行校对和风格统一。

### 4. 自查

快速检查：变量是否漏判、术语是否一致。有一处不一致就修正，无需逐条输出 checklist。

### 5. 写入文件

将最终翻译结果写入当前目录下的 `AI翻译文本.json`，每一条翻译内容最后必须有英文逗号结尾，每个类别最后添加一条空的翻译结果（普通/长文本用 `"": ""`，正则用 `["", ""]`），文件不存在则自动创建。已有文件时，新条目插入到对应类别区段内。
